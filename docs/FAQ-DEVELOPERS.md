# Developer FAQ

This document answers common questions from developers evaluating or integrating the First Step Technology LLC Compliance-as-Code Platform.

---

# Is this repository the commercial product?

No.

This repository contains public documentation, examples, schemas, architecture documentation, and integration guidance.

Commercial Compliance-as-Code rule libraries are licensed separately.

---

# Can I use the JSON Schemas?

Yes.

The published JSON Schemas are intended to help developers:

- Validate data
- Build integrations
- Learn the platform
- Test applications
- Prototype solutions

---

# Can I modify the schemas?

Yes.

Organizations may extend the schemas for internal use.

Maintaining compatibility with the published schemas is recommended for future interoperability.

---

# Are the examples production-ready?

No.

The examples are intentionally simplified for learning and demonstration purposes.

Production implementations should include:

- Error handling
- Logging
- Authentication
- Authorization
- Testing
- Monitoring
- Version management

---

# Can I integrate with existing GRC platforms?

Yes.

The platform is designed to integrate with many governance and compliance platforms, including:

- ServiceNow
- AuditBoard
- Archer
- MetricStream
- Jira

---

# Which programming languages are supported?

Any language capable of processing JSON can integrate with the platform.

Common examples include:

- Python
- JavaScript
- TypeScript
- Java
- Go
- C#
- PowerShell
- Rust

---

# Can AI assistants use the platform?

Yes.

The platform is designed to support AI-assisted compliance workflows including:

- Compliance search
- Control explanations
- Report generation
- Evidence analysis
- Governance automation
- Intelligent recommendations

---

Public API documentation is included in this repository.

Commercial API services are planned for future releases.
