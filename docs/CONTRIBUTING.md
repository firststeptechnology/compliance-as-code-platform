# Contributing

Thank you for your interest in the First Step Technology LLC Compliance-as-Code Platform.

This repository exists to provide public documentation, schemas, developer examples, and integration guidance for the Compliance-as-Code Platform.

We welcome constructive contributions that improve the quality of the public documentation and developer experience.

---

# Ways to Contribute

Contributions are welcome in areas including:

- Documentation improvements
- Typographical corrections
- Example enhancements
- JSON Schema improvements
- Integration documentation
- API documentation
- Architecture documentation
- Developer tooling
- Tutorials
- Educational content

---

# Before Contributing

Please:

- Read the documentation
- Review existing Issues
- Search existing Pull Requests
- Follow repository standards
- Keep changes focused
- Test examples before submitting

---

# Documentation Standards

Documentation should:

- Use Markdown
- Be written in clear English
- Include descriptive headings
- Include examples where appropriate
- Link to related documentation
- Remain technically accurate

---

# Example Standards

Developer examples should:

- Be simple
- Be readable
- Include comments when appropriate
- Demonstrate best practices
- Avoid unnecessary complexity
- Be easy to reproduce

---

# JSON Standards

JSON examples should:

- Validate against published schemas
- Use UTF-8 encoding
- Be consistently formatted
- Use descriptive property names
- Avoid unnecessary nesting

---

# Pull Request Guidelines

Please ensure that Pull Requests:

- Have a clear description
- Address a single topic
- Update documentation if needed
- Follow repository formatting
- Avoid unrelated changes

---

# Issues

GitHub Issues may be used for:

- Documentation improvements
- Bug reports
- Example corrections
- Enhancement suggestions
- Feature requests

---

# Commercial Content

Please do not submit:

- Proprietary rule libraries
- Licensed framework content
- Customer information
- Confidential data
- Commercial datasets
- Internal enterprise materials

---

By participating in this repository, contributors agree to follow the project's Code of Conduct.
