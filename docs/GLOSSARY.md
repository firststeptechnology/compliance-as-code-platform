# Compliance-as-Code Glossary

This glossary defines common terms used throughout the First Step Technology LLC Compliance-as-Code Platform documentation.

---

# AI

Artificial Intelligence. Computer systems that perform tasks typically requiring human intelligence, including reasoning, prediction, and decision support.

---

# API

Application Programming Interface. A standardized method for software applications to communicate with one another.

---

# Assessment

The process of evaluating an organization's implementation of compliance, security, privacy, or governance requirements.

---

# CI/CD

Continuous Integration and Continuous Delivery (or Deployment). Automated processes used to build, test, validate, and release software.

---

# Compliance-as-Code

The practice of representing compliance requirements as structured, machine-readable data that can be validated, automated, searched, and integrated into software systems.

---

# Control

A specific security, privacy, or governance requirement within a framework.

Examples include:

- AC-2
- IA-5
- SC-7

---

# DevSecOps

The integration of security practices throughout the software development lifecycle.

---

# Evidence

Documentation, configurations, logs, reports, screenshots, or other artifacts demonstrating implementation of a compliance requirement.

---

# Framework

A published collection of controls, requirements, or best practices.

Examples include:

- NIST SP 800-53 Rev. 5
- NIST SP 800-171 Rev. 2
- CMMC Level 2
- HIPAA Security Rule
- CCPA / CPRA

---

# GRC

Governance, Risk, and Compliance.

A discipline focused on aligning organizational objectives, managing risk, and meeting regulatory obligations.

---

# JSON

JavaScript Object Notation.

A lightweight, machine-readable data format commonly used for APIs, automation, and structured data exchange.

---

# JSON Schema

A standard for describing and validating the structure of JSON documents.

---

# Machine-Readable

Information structured so software applications can process it automatically without manual interpretation.

---

# Metadata

Information describing another piece of information.

Examples include:

- Framework
- Version
- Control Family
- Status
- Priority

---

# Rule Library

A structured collection of machine-readable compliance rules organized by framework.

Commercial rule libraries are proprietary products of First Step Technology LLC.

---

# Schema

A formal definition describing the required structure, data types, and validation rules for data objects.

---

# Semantic Versioning

A versioning approach using the format:

```
MAJOR.MINOR.PATCH
```

Example:

```
1.0.0
```

---

# Validation

The process of confirming that data conforms to an expected structure or rule set.

---

# Version Control

The practice of tracking changes to files over time using systems such as Git.
