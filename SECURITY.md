# Security Policy

Thank you for helping improve the security of the First Step Technology LLC Compliance-as-Code Platform.

We take responsible security disclosure seriously and appreciate reports that help us improve the security of our software, documentation, and supporting resources.

---

# Supported Versions

The public repository is actively maintained.

Security updates are provided for the latest supported version of the repository documentation and associated public resources.

---

# Reporting a Security Vulnerability

If you discover a security vulnerability, please report it responsibly.

Please include, when possible:

- Description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)
- Supporting screenshots or logs
- Proof of concept (when appropriate)

Please do **not** publicly disclose a vulnerability until it has been reviewed.

---

# What to Report

Examples include:

- Authentication issues
- Authorization weaknesses
- Sensitive data exposure
- Dependency vulnerabilities
- Insecure default configurations
- API security concerns
- Documentation security issues
- Supply chain concerns

---

# Response Process

After receiving a report, we will:

1. Acknowledge receipt.
2. Review the report.
3. Assess severity and impact.
4. Validate the findings.
5. Develop and test a fix, when applicable.
6. Publish updates as appropriate.

Response times may vary depending on complexity and impact.

---

# Scope

This policy applies to:

- Public documentation
- JSON Schemas
- Example code
- API specifications
- Integration documentation
- Repository configuration

Commercial products and customer deployments may follow separate support agreements.

---

# Responsible Disclosure

Please avoid:

- Accessing data that does not belong to you.
- Disrupting services or availability.
- Social engineering.
- Automated denial-of-service testing.
- Actions that violate applicable laws or regulations.

---

# Security Best Practices

Developers using this repository are encouraged to:

- Keep dependencies up to date.
- Validate all external input.
- Protect sensitive credentials.
- Follow the principle of least privilege.
- Use secure coding practices.
- Perform regular security testing.
